\hypertarget{README.md_autotoc_md58}{}\doxysubsubsection{\texorpdfstring{Repository structure}{Repository structure}}\label{README.md_autotoc_md58}

\begin{DoxyItemize}
\item The root of this repository contains the three files that are common to every port -\/ \doxylink{list_8c}{list.\+c}, \doxylink{queue_8c}{queue.\+c} and \doxylink{tasks_8c}{tasks.\+c}. The kernel is contained within these three files. \doxylink{croutine_8c}{croutine.\+c} implements the optional co-\/routine functionality -\/ which is normally only used on very memory limited systems.
\item The {\ttfamily ./portable} directory contains the files that are specific to a particular microcontroller and/or compiler. See the readme file in the {\ttfamily ./portable} directory for more information.
\item The {\ttfamily ./include} directory contains the real time kernel header files.
\item The {\ttfamily ./template\+\_\+configuration} directory contains a sample {\ttfamily Free\+RTOSConfig.\+h} to help jumpstart a new project. See the \href{examples/template_configuration/FreeRTOSConfig.h}{\texttt{Free\+RTOSConfig.\+h}} file for instructions.

\hypertarget{README.md_autotoc_md62}{}\doxysubsubsubsection{\texorpdfstring{Spelling and Formatting}{Spelling and Formatting}}\label{README.md_autotoc_md62}
We recommend using \href{https://code.visualstudio.com}{\texttt{Visual Studio Code}}, commonly referred to as VSCode, when working on the Free\+RTOS-\/\+Kernel. The Free\+RTOS-\/\+Kernel also uses \href{https://cspell.org/}{\texttt{c\+Spell}} as part of its spelling check. The config file for which can be found at \href{cspell.config.yaml}{\texttt{cspell.\+config.\+yaml}} There is additionally a \href{https://marketplace.visualstudio.com/items?itemName=streetsidesoftware.code-spell-checker}{\texttt{c\+Spell plugin for VSCode}} that can be used as well. {\itshape \href{.github/.cSpellWords.txt}{\texttt{.c\+Spell\+Words.\+txt}}} contains words that are not traditionally found in an English dictionary. It is used by the spellchecker to verify the various jargon, variable names, and other odd words used in the Free\+RTOS code base are correct. If your pull request fails to pass the spelling and you believe this is a mistake, then add the word to {\itshape \href{.github/.cSpellWords.txt}{\texttt{.c\+Spell\+Words.\+txt}}}.
