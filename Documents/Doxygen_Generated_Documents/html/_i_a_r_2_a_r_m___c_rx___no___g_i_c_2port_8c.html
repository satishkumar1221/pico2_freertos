◆ ulCriticalNesting

volatile uint32_t ulCriticalNesting = 9999UL

Variable used to keep track of critical section nesting.

This variable is stored as part of the task context and must be initialised to a non zero value to ensure interrupts don't inadvertently become unmasked before the scheduler starts. As it is stored as part of the task context, it will be set to 0 when the first task is started.
